#Branch Configuration

* production - This branch serves as the live version of the application. Working version of the app.
* master - Stores working versions of the application, before being deployed to production. It cannot be a broken version of the app.
* development - Stores the working copy / devleopment version of our application, before deployment to staging. It may be broken, and this is the branch we will all merge from.
